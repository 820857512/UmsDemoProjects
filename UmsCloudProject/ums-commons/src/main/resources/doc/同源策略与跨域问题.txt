1、对于浏览器来说，如果要进行异步请求，则请求的地址包含：IP地址、端口号和项目应用名必须保持一致，
    也就是获得该 HTML 页面的 IP 地址、端口号和项目应用名必须和该页面所发出异步请求的 IP 地址、端口号和项目应用名必须保持一致。
    否则浏览器会认为该异步请求不安全，会禁止发送本次异步请求。称之为浏览器的同源策略。
2、解决跨域问题的方案有多种，第一种在控制层增加注解：@CrossOrigin；第二种使用反向代理来完成跨域。
3、通用的反向代理会使用 Nginx 来完成。
4、Nginx 服务器有三种功能：1、Web 应用服务器（类似于 Tomcat）；2、反向代理服务器；3、负载均衡。
